# Security Policy

## Reporting a Vulnerability

The $EVERYTHING community takes security seriously. If you discover a vulnerability, please report it responsibly.

### How to Report

**DO NOT** open a public issue for security vulnerabilities.

Instead, please email: **security@everything.meme**

Or DM the team on [Twitter](https://twitter.com/everythingcoin).

### What to Include
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response Timeline
- **Acknowledgment:** Within 48 hours
- **Assessment:** Within 1 week
- **Fix/Resolution:** Depends on severity

### Scope

In scope:
- Website (everything.meme)
- Smart contract interactions
- Community tooling & bots

Out of scope:
- Third-party services (DEXs, block explorers)
- Social engineering attacks
- Issues already reported

## Smart Contract

The $EVERYTHING token contract is:
- **Renounced** — No owner can modify it
- **Audited** — Reviewed for common vulnerabilities
- **Immutable** — No proxy pattern, no upgrades

The contract source code is available in `/contracts` for transparency.

## Bug Bounty

We appreciate security researchers who help keep $EVERYTHING safe. Significant findings may be rewarded with $EVERYTHING tokens at the community's discretion.
